================================================================================
                            📊 PROJECT SUMMARY
================================================================================

FinSight AI is a complete, production-quality Python application that:
- Accepts receipt images from users
- Extracts text using Tesseract OCR
- Parses merchant names and amounts using Llama3 LLM
- Automatically categorizes expenses
- Stores expenses in SQLite database
- Analyzes spending patterns and trends
- Generates AI-powered cost-saving recommendations

Technology Stack:
  • Backend: Python + FastAPI
  • Frontend: Streamlit
  • Database: SQLite + SQLAlchemy
  • LLM: Ollama running Llama3
  • OCR: Tesseract
  • Workflow: LangGraph
  • API: REST with automatic documentation

                        🚀 QUICK START GUIDE
================================================================================

Prerequisites:
  • Python 3.10+
  • Tesseract OCR
  • Ollama + Llama3
  • 4GB+ RAM

Installation (macOS/Linux):
  1. chmod +x setup.sh
  2. ./setup.sh

Installation (Windows):
  1. setup.bat

Running the Application:
  Terminal 1: ollama serve
  Terminal 2: cd backend && source venv/bin/activate && python main.py
  Terminal 3: cd frontend && source venv/bin/activate && streamlit run app.py

Access Points:
  • API Documentation: http://localhost:8000/docs
  • Dashboard: http://localhost:8501
  • Ollama API: http://localhost:11434

================================================================================
                        🔧 API ENDPOINTS
================================================================================

Expense Endpoints:
  POST   /api/expenses/upload           Upload and process receipt
  POST   /api/expenses/                 Create expense
  GET    /api/expenses/                 List all expenses
  GET    /api/expenses/{id}             Get expense details
  PUT    /api/expenses/{id}             Update expense
  DELETE /api/expenses/{id}             Delete expense

Insights Endpoints:
  GET    /api/insights/spending-summary Get spending summary
  GET    /api/insights/by-category      Get spending by category
  GET    /api/insights/trends           Get spending trends
  POST   /api/insights/recommendations  Get recommendations

Health Endpoint:
  GET    /api/health/                   System health check

                        💾 DATABASE SCHEMA
================================================================================

expenses table:
  • id (int, primary key)
  • merchant_name (varchar, indexed)

  • category (varchar, indexed)
  • date (datetime, indexed)
  • description (text)
  • image_path (varchar)
  • confidence_score (float)
  • created_at (datetime, indexed)
  • updated_at (datetime)

Indexes optimized for:
  - Finding expenses by category
  - Finding expenses by date
  - Searching by merchant name
  - Recent transactions
