KEY DESIGN PATTERNS IMPLEMENTED
─────────────────────────────────────────────────────────────────────────────

1. Repository Pattern
   - ExpenseRepository encapsulates all database queries
   - CRUD operations and rich query methods

2. Service Layer
   - ReceiptService, ExpenseService, InsightService
   - Orchestrate agents and repositories

3. Agent Architecture
   - LLMAgent base class with specialized subclasses
   - CategorizerAgent, AnalyzerAgent, RecommenderAgent

4. Dependency Injection
   - FastAPI Depends() for database sessions
   - Clean separation of concerns

5. Prompt Registry
   - Centralized prompt templates in /src/prompts/
   - get_prompt() and format_prompt() utilities

6. Type Safety
   - Pydantic models throughout
   - Type hints on all functions

7. State Machine (LangGraph)
   - ReceiptProcessingState TypedDict
   - Multi-step workflow with error handling

8. Middleware Pattern
   - CORS middleware in FastAPI
   - Error handling at each layer
